About the Project

Lie That Tells the Truth 2.0

A digital humanities project on Korean women-oriented web novels, romance fantasy clichés, and hashtag networks as structures of desire.

Project Overview

One of the most common criticisms of romance fantasy web novels is their reliance on clichés. Hashtags such as #competentheroine, #girlcrush, #kindmalelead, #regretfulmalelead, #regression, and #contractmarriage may seem to point to the repetitive reproduction of standardized characters and plot structures. These coded elements often make romance fantasy web novels appear less like “literature” and more like commercial products designed to provide readers with immediate catharsis.

This project, however, begins from the opposite premise: romance fantasy may be realistic precisely because of its clichés. A cliché is not merely an old or repetitive narrative formula. It is also a condensed form of desire, repeatedly selected, consumed, and reproduced by readers. In romance fantasy web novels, hashtags function as codes that break down stories into smaller elements, such as the female protagonist, the male protagonist, relationships, conflicts, narrative devices, and forms of reward. Readers use these codes to find the stories they want, platforms use them to classify works, and writers and markets recombine them to produce new stories.

From the perspective of digital humanities, these hashtags can be understood not simply as search terms, but as meaningful units for distant reading. The web novel market operates through continuous reader engagement, recurring payments, and immediate audience response. For this reason, hashtags do more than describe the content of individual works. They can also be read as indicators of what kinds of characters, relationships, and narrative rewards readers desire.

Lie That Tells the Truth 2.0 is a digital humanities project that analyzes the relationships among hashtags used in contemporary Korean women-oriented web novels and examines the desires and gendered imaginations embedded within them. In women-oriented web novels, a single hashtag rarely produces meaning on its own. Rather, multiple hashtags are used together to construct a specific character type, relationship dynamic, or narrative structure. Through hashtag network analysis, this project explores how the repeated codes of romance fantasy reveal the desires, anxieties, expectations, and idealized subjectivities of contemporary women readers.

This approach does not replace close reading with distant reading. Instead, it places the two methods in a complementary relationship. While close reading allows us to examine the meanings of characters and narratives within individual texts, hashtag network analysis reveals recurring patterns and structures of desire across a larger body of works. By connecting these two modes of reading, this project argues that romance fantasy web novels are not merely escapist fantasies. Rather, they are “lies that tell the truth,” fictional worlds that reveal how contemporary Korean women readers imagine, negotiate, and respond to the realities of gender, power, and social institutions.

Dataset

The dataset was built from a collection of Korean romance fantasy webnovels and their associated platform hashtags. Each work was assigned a unique work ID, and each hashtag attached to that work was recorded as a separate work-tag relationship.

works.csv

Contains work-level information, such as work ID, title, ranking information, platform, publication form, and brief descriptive fields when available.

work_tags.csv

Contains the relationship between each work and each hashtag. This file is the main source for calculating hashtag frequency and co-occurrence.

tag_metadata.csv

Contains manually assigned hashtag categories, such as female protagonist, male protagonist, genre or plot, and hybrid.

Data Preparation

The original dataset was reorganized into a long-format structure so that each row in work_tags.csv represents one relationship between a work and a hashtag. Hashtag symbols were removed from the raw tag strings for consistency, and missing or non-informative values were excluded from the network.

Hashtags were then categorized into broad analytical groups. These categories are not automatic classifications. They reflect interpretive decisions made for this project in order to compare how different kinds of tags operate within the same platform system.

Network Methodology

In this network, each node represents a hashtag. An edge is created when two hashtags appear in the same work. The weight of an edge represents the number of works in which the two hashtags appear together.

Node

A hashtag, such as 능력녀, 걸크러쉬, or 회귀물.

Edge

A co-occurrence relationship between two hashtags within the same work.

Frequency

The number of works in which a hashtag appears.

Edge weight

The number of works in which two hashtags appear together.

The visualization was built with D3.js. The graph is calculated from the source CSV files in the browser, which allows the user to filter and compare the network interactively.

How to Use the Network Tool

  • Use category filters to show or hide broad hashtag groups.
  • Use minimum frequency to remove hashtags that appear in fewer works.
  • Use minimum edge weight to remove weak co-occurrence relationships.
  • Use Hide Tags to remove dominant tags such as 능력녀 or 로맨스판타지.
  • Use Compare Tags to compare two or more selected hashtags.
  • Hover over a node to see its category, frequency, and strongest connections.
  • Drag a node to pin it in place, and double-click to unpin it.

Limitations and Data Ethics

This project does not claim that hashtags fully represent the content of a work. Platform tags are shaped by discoverability, marketing, reader expectations, and genre convention. They are useful cultural data, but they should not be treated as neutral or complete descriptions of the text itself.

The category labels used in this project also involve interpretation. Some hashtags can belong to multiple conceptual groups, and some may shift meaning depending on platform context or reader usage. For this reason, the network should be read as an exploratory map rather than a final classification of the genre.
